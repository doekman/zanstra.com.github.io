IE Mac documentation

Windows people: Internet Explorer is also available for Macintosh (and Unix). However, it's a seperatly developed application. Because of this, some features available on Windows are not available on Macintosh, and vice-versa.

Standards

This is not good. We are supposed of living in a world of world of standards. But OK, so you would like to lookup the differences in a reference. MSDN Library is such a reference.

Here it goes bad. Although MSDN is pretty accurat on Windows, the info supplied on the Macintosh version of Internet Explorer is poor. The data on the HTML part is available, but often incorrect. Data on the javascript language on Macintosh is completely absent. That's why this project was started.

Action

If you find inconsistencies in MSDN, or discover a javascript (or ECMAScript) feature is not available on Mac, or is introduced in version x.y, please mail me. Preferably in the XML format I defined.

Please don't mail me about: "I can't script SVG." You can mail Uncle Bill about that (or even better: help the guys from Mozilla/SVG). This page is about:

  • Flaws in MSDN Library's DHTML Reference, and
  • No entry on on Mac's javascript implementation at JScript version information.
